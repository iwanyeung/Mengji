# 梦悸 产品需求文档（PRD）v1.0

## 0. 概要

| 项目 | 说明 |
|------|------|
| **产品名称** | 梦悸（英文暂用：Dream Impulse） |
| **一句话定义** | 一款以梦境记录为入口、以温柔陪伴式 AI 解读为桥梁、以视觉化内容生成为核心价值的**灵感创作工具**。 |
| **核心价值主张** | 帮你在醒来后**留住梦**、**看懂自己**，并把这些转瞬即逝的画面与情绪显化成**漫画、插画和短动画**，成为个人灵感档案。 |

---

## 1. 产品背景与机会

### 1.1 用户痛点

- **梦境易逝**：醒来几分钟内，梦内容快速消散，难以复盘与对比。
- **记录门槛高**：传统日记需要打字/写字，醒来时常没耐心、没精力。
- **内容零碎难回看**：偶尔记下的梦大多是散乱文本和截图，后续无法检索和关联。
- **灵感浪费**：创作者常在梦中获得画面、主题或情绪灵感，但缺乏系统化的收集工具。
- **解读需求存在但不敢信**：市面上“解梦”内容往往要么太玄学、要么太硬核学术，用户既有兴趣又不完全信任。

### 1.2 市场机会

- Dream journaling / 睡眠应用在海外已有一定规模，但**中文市场缺乏“高审美 + AI + 创作导向”的梦境应用**。
- 大模型和生成式 AI 技术已可支撑：高质量中文 ASR 转写与文本整理、个性化解释文案、文生图/文生动画。
- 创作者与精神自我探索人群对“私密 + 美感 + 有趣”的工具有持续需求，且付费意愿相对较高。

### 1.3 产品定位

- **主定位**：灵感创作工具。
- **次价值**：情绪与自我观察、温柔陪伴式“梦后回看”。
- **明确非定位**：非医疗产品；非心理诊断工具；非大众星座/玄学解梦类娱乐产品。

---

## 2. 目标用户与使用场景

### 2.1 目标用户画像

**核心用户（优先满足）**

- 20–40 岁城市用户，创意/内容相关职业或爱好者：设计师、插画师、摄影师、编剧、作家、学生等。
- 对自我观察、心理学、灵性/潜意识话题感兴趣。
- 使用过或好感于 Notion、Day One、Calm、VSCO 等审美感较强的产品。

**扩展用户**

- 有一定焦虑或情绪困扰，希望通过记录梦境了解自己的人。
- 睡眠不稳定，想更关注自己夜间状态的人。

### 2.2 典型使用场景

| 场景 | 描述 |
|------|------|
| **刚醒来** | 用户睁眼后半梦半醒，打开梦悸，长按录音按钮，快速把梦讲一遍；稍后在地铁/通勤路上打开“梦析”页，看到 AI 整理的文字和温柔的解读。 |
| **创作前** | 用户准备画画/写作，打开“潜意识星图”，寻找最近反复出现的意象（如“楼梯”“牙齿”“水下城市”），选择一条梦，通过“显化工坊”生成四格漫画或一页灵感卡，作为创作起点。 |
| **长期自我观察** | 用户每周回顾一周的梦，注意到反复的情绪（如紧张、追赶、迷路）；梦悸给出温柔提示：最近的压力点、经常出现的情绪、与睡眠时长/入睡时间的粗略关联。 |

## 3. 产品目标与成功指标

### 3.1 MVP 阶段目标（iOS App，仅简体中文）

- 完成**录音 → 转写 → 整理 → 解读 → 存档**的完整闭环。
- 提供基础版“潜意识星图”，展示梦境之间的关联。
- 打通首个付费点：**四格漫画生成**。
- 建立清晰品牌审美：**超现实艺术风 + 杂志感排版 + 夜色基调**。

### 3.2 核心 KPI（MVP）

- 新用户首日**至少完成 1 条梦境记录**的比例。
- 单用户**7 日内记录梦境次数**（活跃深度）。
- 从“梦析页”点击进入“显化工坊”的比例。
- 四格漫画生成的付费转化率。
- 7 日/30 日留存（粗指标）。

## 5. 详细功能需求（MVP）

### 5.1 录梦页（Tab：录梦，对应原型：虚空）

#### 5.1.1 目标

- 降低记录门槛，让用户醒来 3 秒内可以开始录。
- 支持分段录制 + 上滑锁定，适合长梦与零碎片段。
- 记录后无缝衔接 AI 转写与梦析。

#### 5.1.2 页面元素

- 顶部出血标题：“告诉我”（可沿用原型视觉）。
- 左侧近期梦境碎片列表（文字预览 + 日期时间）。
- 右下角主录音按钮：
  - 默认状态：酸性黄圆形按钮 + 波纹边缘（“按住录音”）。
  - 录音中：按钮变为红色光晕，背景/声波动效。

#### 5.1.3 交互逻辑

- **按住录音**：按下即开始录制，显示录制时长与声波动效；允许用户说完松手即结束一段。
- **上滑锁定**：按住后向上滑动，进入“锁定录音模式”，无需持续按压；再次点击结束。
- **分段录制**：每段结束后，立即在页内列表显示该段录音时长与该段 AI 转写文字；用户可继续录下一段。
- **完成录制**：用户点击“完成并整理”后，将所有片段合并发送服务端进行梦境整理与梦析准备，并跳转至“梦析”页。

#### 5.1.4 与 Apple Watch / 小程序的关系（MVP 声明）

- MVP **暂不实现 Watch 与小程序联动**。
- 技术上预留接口：录音来源字段（iPhone / Watch / MiniProgram），未来扩展时兼容。

---

### 5.2 梦析页（Tab：梦析，对应原型：神谕）

#### 5.2.1 目标

- 提供**文学化整理 + 温柔陪伴式解读 + 关键词可视化**。
- 作为显化工坊的自然入口（“将这个梦显化为作品”）。

#### 5.2.2 页面结构

- 顶部导航：返回 / 更多（导出、删除、标星）。
- 左侧/上方：日期与时间。
- **主体上半部分**：梦境整理文本  
  - AI 将片段整理成 1–2 段流畅、略带文学性的中文叙述；语气平实 + 诗意；支持关键词高亮（人、地点、情绪、物件）。
- **中部**：关键词与标签  
  - 主题标签（如 `[重复场景] [追逐] [飞行]`）、情绪标签、元素标签；UI：0 圆角，酸性黄描边胶囊。
- **下半部分**：温柔陪伴式解读文本  
  - 情绪观察、主题与模式、自我对话建议；后期可结合睡眠数据给出温和提示。
- **底部 CTA**：主按钮“显化为四格漫画（付费）”；次文案“稍后再说，先把它留在潜意识星图里”。

#### 5.2.3 科学与合规边界（文案级）

- 在设置或梦析页底部显示简短声明：  
  “梦悸提供的梦境整理和解释仅用于自我观察与灵感启发，不构成医学或心理诊断建议。如有持续的情绪困扰，请考虑咨询专业机构。”

---

### 5.3 显化工坊（Tab：显化工坊，对应原型：显化（视觉））

> MVP 重点：**四格漫画生成功能上线，可付费**。一页纸与 10 秒动画定义在 PRD 中，实现排入后续版本。

#### 5.3.1 目标

- 将抽象梦境显化为可保存、收藏、分享的视觉作品。
- 成为核心变现场景。

#### 5.3.2 流程（四格漫画 MVP）

1. **进入方式**：从某条梦的“梦析页”点击“显化为四格漫画”，或在显化工坊页选择“从历史梦境中挑选一条”。
2. **选择梦境**：默认选中最近一次已解析梦境；支持从列表中选择最近 N 条梦（仅已完成梦析的）。
3. **选择样式**：MVP 可固定 1–2 种风格（如高对比黑白 + 半调颗粒、霓虹色超现实拼贴等），用文案描述风格。
4. **确认并付费**：单次购买；若仍为游客，先完成 Apple / 微信登录再下单。
5. **生成与展示**：生成成功后展示四格漫画面板，支持保存到本地相册，并自动绑定到该梦记录，在潜意识星图中以“已显化”标识。

#### 5.3.3 未来扩展（PRD 定义，非 MVP 必做）

- **一页纸**：将梦境浓缩成一张海报式图片（适合手机壁纸/打印）。
- **10 秒动画**：以关键意象为主的短循环动画（MP4 / GIF）。

---

### 5.4 潜意识星图（Tab：潜意识星图，对应原型：潜意识档案馆）

#### 5.4.1 目标

- 把线性“梦境历史”转化为**非线性、星空式的“潜意识地图”**。
- 成为产品在审美与体验上的核心差异点。

#### 5.4.2 展示原则（MVP）

- 中心区域 2D 画布：以节点显示梦境。
- 每个节点信息：日期（如 10.24）、标签简写（如 `[牙齿] [追逐]`）、开头一句（截断）、是否已生成视觉作品（小图标/边框高亮）。
- 节点布局：以时间为粗参考轴（近的靠中心，远的靠边缘），加随机偏移，营造星空散布感。
- **连接线**：当两个或多个梦在情绪、意象、人物、地点等维度上高度相似时，用呼吸感线条连接；明暗度或粗细与“关联强度”挂钩。

#### 5.4.3 交互 MVP

- 拖拽平移画布；基础缩放（支持双指缩放），并提供“重置视图”恢复默认平移与缩放。
- 点击节点：弹出梦境卡片（日期 + 关键词 + 一句摘要 + 是否有显化作品），点击“查看详情”跳转到对应“梦析页”。
- 底部搜索/过滤：文本搜索（梦境全文关键词）；简单过滤（只看“已显化”、只看“含某标签”）。

## 6. 用户账号与登录

### 6.1 游客模式

- 打开 App 即可使用录梦和基本梦析功能（数据以本机匿名 ID 暂存）。
- 限制：生成四格漫画前必须登录或绑定；若用户卸载 App 或系统清理数据，游客数据可能丢失，需在首次引导中明确提示。

### 6.2 登录方式

- **Apple 登录**、**微信登录**。
- 登录后：将本机游客数据合并绑定到该账号；后续可在多设备（未来）同步。

## 7. AI 能力与服务架构（概述）

### 7.1 模块拆分

| 模块 | 说明 |
|------|------|
| ASR（语音转写） | 中英文混合识别，支持分段转写。 |
| 梦境整理（文本重写） | 从口语碎片 → 连贯的中文文学化文本（保持内容忠实）。 |
| 关键词与标签提取 | 人物、地点、物件、情绪、场景类型。 |
| 启发式梦析（文本生成） | 温柔陪伴口吻，给出可能的情绪与主题理解，不做诊断。 |
| 关联计算 | 基于关键词、情绪标签、时间距离，计算梦境间相似度，用于潜意识星图。 |
| 显化工坊提示词生成 | 从梦文本中抽取视觉元素、风格线索，拼装成文生图/文生动画 Prompt。 |

### 7.2 技术约束（高层）

- AI 调用使用 **See Dance 接口**，后端部署在 **腾讯云**。
- 所有 AI 调用均通过服务端完成，客户端不直接持有密钥。
- 原始语音、原始转写、整理稿、解读稿全部以结构化形式存储，支持后期迁移/重跑。

## 8. 科学依据与风险边界（PRD 正式条款）

- 本应用基于当代梦研究的一般共识，认为：梦与记忆整合、情绪调节及个体关切主题相关；长期记录梦境有助于提升自我觉察与情绪反思能力。  
  参考：Cleveland Clinic (2024)、Nir & Tononi (2010)。
- 同时明确：
  - 单次梦境无法成为心理诊断或医疗判断依据。
  - 本应用的“梦析”仅为启发式、自我观察与灵感参考。
  - 对自伤、自杀、极端绝望等高风险内容，产品需在文案与流程上引导用户寻求专业帮助。

该条款将以简化文案形式出现在：注册流程、设置页“关于与声明”、梦析页底部。

## 9. 版本规划

### 9.1 MVP（v1.0）

- iOS App（简体中文）。
- 游客记录 + Apple / 微信登录。
- 录梦页（分段录制 + 转写）、梦析页（整理 + 标签 + 温柔陪伴式解读）。
- 潜意识星图基础版（节点 + 基础关联线）。
- 显化工坊：四格漫画生成（付费）。
- 基础埋点（录梦次数、梦析访问、显化点击与支付转化）。

### 9.2 v1.1 – 睡眠与 Watch

- Apple Watch 录音入口；HealthKit 睡眠数据接入；梦析页中加入简单的睡眠背景提示。

### 9.3 v1.2 – 微信小程序

- 查看梦境与梦析；查看已生成四格漫画/一页纸；基础分享与引导跳转 App。

### 9.4 v2.x – 创作增强

- 一页纸生成；10s 动画生成；年度潜意识回顾报告；更多艺术风格模板。

## 10. 设计系统（设计令牌）

与 `超现实艺术风 (Surreal & Artistic).pdf` 一致，开发时请统一使用以下令牌：

```css
:root {
  --color-primary: #D4FF33;      /* 酸性黄 - CTA、录音激活态 */
  --color-background: #0D0C0F;  /* 漆黑 - 基础页面 */
  --color-surface: #1E1A25;     /* 深紫 - 卡片、悬浮面板 */
  --color-text: #F4F0EB;       /* 骨白 - 正文、标题 */
  --color-muted: #7A7585;      /* 灰烬 - 元数据、边框、时间戳 */
  --color-accent: #FF3366;     /* 迷幻粉 - 警告、重复模式高亮 */
  --font-display: 'Bodoni Moda', serif;
  --font-body: 'Space Grotesk', sans-serif;
  --radius-none: 0px;
  --radius-pill: 999px;
}
```

- 排版：标题 Bodoni Moda（可选斜体）；副标题/正文 Space Grotesk；按钮 Space Grotesk 大写。
- 样式原则：硬边缘（0px 圆角）、细 1px 边框、背景颗粒噪点叠加、粗野主义 + 杂志感。
